Auth email templates for Supabase

Auth email templates

The HTML for the four Supabase auth email templates — Confirm signup, Password recovery, Magic link, and Email change. Each block has a Subject and a Body (HTML) field. Use the copy buttons to grab the value, then paste it into Auth → Email Templates in your Supabase dashboard.

Setup context: these templates are designed for Supabase sending email via Custom SMTP → Resend (configured in Settings → Auth → SMTP). The {{ .ConfirmationURL }} variable is the magic link Supabase generates — click it and the user lands on the page configured in Site URL / Redirect URLs. We use https://recalleducation.co.uk as the base.
Confirm signup signup
Open in dashboard
Where it goes: Supabase → Auth → Email Templates → Confirm signup. Used when: a new user signs up with email/password and "Confirm email" is enabled.
Available template variables
{{ .ConfirmationURL }} — magic link (use this in the CTA button)
{{ .Token }} — 6-digit OTP, also valid as a code
{{ .TokenHash }} — hashed token (used internally)
{{ .SiteURL }} — Site URL from your dashboard config
{{ .Email }} — the user's email address
{{ .Data }} — full user_metadata blob (from signUp options.data)
Password recovery recovery
Open in dashboard
Where it goes: Supabase → Auth → Email Templates → Reset password. Used when: a user requests a password reset (the link goes to /reset-password.html).
Available template variables
{{ .ConfirmationURL }} — recovery link (lands on /reset-password.html)
{{ .Token }} — 6-digit OTP
{{ .SiteURL }} — Site URL
{{ .Email }} — the user's email
Magic link magiclink
Open in dashboard
Where it goes: Supabase → Auth → Email Templates → Magic link. Used when: a user requests a magic-link sign-in (the link goes to /auth/confirmed.html).
Available template variables
{{ .ConfirmationURL }} — magic link (lands on /auth/confirmed.html)
{{ .Token }} — 6-digit OTP, valid as a sign-in code
{{ .SiteURL }} — Site URL
{{ .Email }} — the user's email
Email change email_change
Open in dashboard
Where it goes: Supabase → Auth → Email Templates → Change email address. Used when: a user updates their email address (rare; only fires if you call auth.updateUser({ email })).
Available template variables
{{ .ConfirmationURL }} — confirmation link for the new address
{{ .Token }} — 6-digit OTP
{{ .Email }} — the NEW email address
{{ .OldEmail }} — the previous email address (if exposed by template)
{{ .SiteURL }} — Site URL
Heads up on the Edge Function: the email templates above are the Supabase dashboard templates — they run when Supabase sends the email itself (default SMTP or Custom SMTP via Resend). They do not run when the send-auth-email Edge Function is in use. If you re-enable the hook later, you'll want to either (a) keep the hook disabled and rely on these templates, or (b) edit the templates inside the Edge Function's index.ts and re-deploy.
